---
title: "Referrals & Rewards"
---

Referral and rewards endpoints provide access to referral information, XP rewards, weekly summaries, and reward history.

These are authenticated user endpoints. Use bearer auth or API-key auth as described in [Onboarding & Auth](./api-reference/integration/onboarding-auth).

## Referral Summary

```http
GET /api/v1/user/referral
```

Returns the current user's referral information.

The response includes:

- `referral_code`
- `referrer_code`
- `referrer_user_id`
- `downline_count`

```json
{
  "referral_code": "EKIDEN123",
  "referrer_code": "PARTNER42",
  "referrer_user_id": "user_abc",
  "downline_count": 12
}
```

## Bind a Referrer

```http
POST /api/v1/user/referral
Content-Type: application/json

{ "referral_code": "PARTNER42" }
```

The response is the updated referral summary.

Do not build flows that repeatedly rebind users unless the backend explicitly allows it.

## Rewards Summary

```http
GET /api/v1/user/rewards
```

Returns the current XP balance, current weekly summary, historical weekly summaries, and recent reward ledger entries.

The response includes:

- `xp_balance`
- `current_week`
- `history`
- `ledger`

```json
{
  "xp_balance": 125000,
  "current_week": {
    "week_start": 1731283200000,
    "week_end": 1731888000000,
    "own_ps": "90000",
    "bonus_ps": "15000",
    "final_ps": "105000",
    "xp": "105000",
    "finalized": false
  },
  "history": [],
  "ledger": []
}
```

See the generated OpenAPI schema (`RewardSummaryResponse`) for the complete response structure.

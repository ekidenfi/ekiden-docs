## Overview

Ekiden allows leverage to be configured independently for each trading symbol.

Changing leverage affects the margin requirements for future position exposure and the estimated liquidation price. Increasing leverage reduces the required margin but increases liquidation risk, while lowering leverage increases the required margin and reduces liquidation risk.

## Endpoint

```http
POST /api/v1/position/set-leverage
```

Updates the leverage setting for a specific trading symbol and sub-account.

### Authentication

```text
Authorization: Bearer <JWT>
Content-Type: application/json
```

## Request Parameters

| Parameter | Type | Description | Required |
| --- | --- | --- | --- |
| `leverage` | string | Leverage value to set (for example, `"10"` for 10× leverage). | Yes |
| `sub_account_address` | string | Sub-account address. | Yes |
| `symbol` | string | Trading symbol (for example, `"BTC-USDCx"`). | Yes |

## Example Request

```json
{
  "leverage": "8",
  "sub_account_address": "0x6160e0768af12ec30112972801aaa8287b42f15a2485ef68cd184a3eeec9dc49",
  "symbol": "BTC-USDCx"
}
```

## Response

```json
{
  "success": true,
  "time": 1783479390120
}
```

A successful response indicates that the leverage update request has been accepted.

## Notes

- Leverage is configured independently for each trading symbol.
- Supported leverage ranges vary by market and can be obtained from `GET /api/v1/market/instruments-info`.
- Changing leverage affects margin requirements and the estimated liquidation price.

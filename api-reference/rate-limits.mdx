# Rate Limits

Ekiden‚Äôs API enforces rate limits to ensure fair usage and maintain system reliability. These limits vary depending on the endpoint type, user authentication, and request method (REST or WebSocket). This page outlines our rate-limiting conventions and provides placeholders where specific values will be configured.

---

## Overview

- Rate limits apply per **IP address** and per **API key**
- Different limits may apply to **public** vs **authenticated** endpoints
- **REST** and **WebSocket** APIs have distinct rate limits

## Rate Limit Errors

If you exceed the allowed rate:

- **HTTP Status Code**: `429 Too Many Requests`
- **Response Header**: `Retry-After: {SECONDS}`

### Error Response Body

```json
{
  "error_code": 1015,
  "error_message": "Rate limit exceeded. Please wait {SECONDS} seconds."
}
```

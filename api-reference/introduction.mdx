# Introduction

The Ekiden API provides low-latency access to Ekiden’s trading infrastructure, combining an off-chain central limit order book (CLOB) with Canton-based on-chain workflows.

Orders are submitted, matched, and executed through Ekiden’s off-chain trading infrastructure. This allows traders and applications to place and manage orders without requiring a wallet interaction for every trading operation.

Canton is used for on-chain workflows such as user registration, trading account creation, and funding operations. These workflows are separate from latency-sensitive order submission and execution.
